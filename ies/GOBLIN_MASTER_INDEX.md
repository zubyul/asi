## KEY CONCEPTS REFERENCE

### Agents
- **Goblin**: Intelligent agent with capability discovery
- **LearningGoblin**: Goblin that learns discovery strategy (Phase 2.1)
- **EvolvingGoblin**: Goblin that learns compositions (Phase 2.1)
- **HierarchicalGoblin**: Goblin organized in teams (Phase 2.2)

### Data & Discovery
- **GadgetStore**: DuckDB-backed capability database
- **Capability**: Discoverable skill or function
- **Gadget**: Raw capability from database
- **Refinement Query**: Type-based, arity-based discovery

### Computation
- **2-Transducer**: Finite state machine (inputâ†’stateâ†’output)
- **TwoTransducer**: Implementation of FSM
- **Wireworld**: Cellular automaton for verification
- **Gate**: Logic operation (CNOT, XOR, etc.)

### Composition
- **Free Monad**: Syntactic computation tree
- **Cofree Comonad**: Semantic observation structure
- **Module**: Free as algebra over Cofree as coalgebra
- **Composition**: Combining capabilities via module action

### Frameworks
- **Hoot**: Orchestration environment for goblins
- **IntegratedGoblinSystem**: 8-phase execution pipeline

### Learning (Phase 2+)
- **Multi-Armed Bandit**: Query selection via RL
- **Thompson Sampling**: Bayesian exploration strategy
- **Composition Learning**: Automatic capability synthesis
- **Type Inference**: Learning capability types
